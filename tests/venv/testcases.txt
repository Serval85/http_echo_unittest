Тест кейсы для:
Общие (обычно проверка при каждом тесте):
    1. Проверка статус кода
    2. Проверка reason

GET:
    1. Запрос без параметров.
    2. Запрос с параметром:
        - id
        - name
        - tag
    3. Запрос с 2\3 параметрами.
    4. Запрос с некорректными параметрами (не существующие id\name\tag)
    5. Запрос с несколькими параметрами одного типа (id and id\ name and name...)
    6. Запрос с несколькими параметрами и не корректным параметром.

POST:
    1. Загрузка файла.
    2. Загрузка без параметра: (можно в одном тесте без всего)
        - id (проверка генерации)
        - name (заполнение через id)
        - tag (пофиг если нету)
     3. Загрузка с именем в кириллице или цифрами (не строка)
     4. Загрузка с именем без расширения (3, 4 в одном - да да 2 тест кейса в одном, но только для скорости)
     5. Загрузка без указания длинны - хитрый request че-то не дает указать не корреутную длинну или без нее.
     6. Загрузка с некорректной длинной - см п.5
     7. Проверка перезаписи (при одинаковом id):
        - name
        - modificationTime
     8. Проверка сохранения Content-Type - def test_post(self):
     9. Также можно скачать обратно и проверить на идентичность файлы... но это уже упоротость для обучения.
